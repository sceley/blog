本地存储

前端
  • HTML5

序言

本地数据库是HTML5中新增的一个功能，使用它可以在客户端本地建立一个数据库，原本必须存在服务器端数据库的内容现在可以直接保存在客户端本地了，减轻服务器端的负担。使回到服务器端为轻，客户端为重的时代。

本地储存

  1. web storage
  2. Web Storage
  3. 本地数据库(SQLLIte本地数据库)
  4. indexedDB数据库

Web storage

session storage(临时保存)

保存数据的方法

sessionStorage.setItem('key', 'value');
sessionStorage.key = "value";

读取数据的方法

let ? = sessionStorage.getItem("key");
let ? = sessionStorage.key;

localStorage

保存数据的方法

localStorage.setItem('key', 'value');
localStorage.key = "value";

读取数据的方法

let ? = localStorage.getItem("key");
let ? = localStorage.key;

localStorage.length数据的条数

localStorage.key(index)将想要的数据的索引号作为index参数传入，可以得到localStorage中与这个索引号对应的数据。

localStorage.clear()清除所有的数据

localStorage.removeItem();

storage事件

window.addEventListener('storage', () => {
}, false);
//storage值变动时发生

本地数据库

步骤

  1. 创建访问数据库的对象
  2. 使用事务处理

使用openDatabase创建数据库对象

let db = openDatabase("database name", "version", "description", "size");

调用transaction执行事务 

transaction((tx) => {
    tx.executeSql(sqlquery, [], dataHandler, errorHandler);
});
//第一条语句为需要执行的SQL语句
//第二条为使用到的参数数组
//第三为执行成功的回调(tx, results) => {

}
//第三为执行失败的回调(tx, err) => {

}

创建数据库表

transaction((tx) => {
    tx.executeSql('create table if not exists logs (id unique, log)');
});

追加数据

tx.executeSql('insert into peron values(?, ?, ?);', []);
或tx.executeSql('insert into person(age, name) values(?, ?)', [])

更新数据

tx.executeSql('update person set age=? where name=?;', [age, name]);

查找数据

tx.executeSql('select * from person ;', []);
tx.executeSql('select * from person where name=?;', []);

删除数据

tx.executeSql('delete from person where id=?';);

indexedDB数据库

兼容各浏览器

window.indexedDB = window.indexedDB || window.webkitIndexedDB || 
window.mozIndexDB || window.msIndexedDB
window.IDBKeyRange = window.IDBKeyRange || window.webkitIDBKeyRange ||
window.mozIDBKeyRange || window.msIDBKeyRange;
window.cursor = window.cursor || window.webkitCursor || 
window.mozCursor || window.msCursor;

连接数据库

indexedDB.open(dbname, dbVersion);
//返回IDBOpenDBRequest对象, 代表数据库连接的请求对象
//dbname为必须参数, dbVersion省略默认为1且修改时值不能比前一次的值小, 否则会报错。

dbConnection.onsuccess = (e) => {
    e.target.result
    // 为一个IDBDatabase对象, 代表连接成功的数据库对象
}
//onsuccess事件和onerror事件来定义数据库的连接状态

idb.close()关闭数据库

数据库的版本更新

事务分为三种：只读事务, 读写事务, 版本更新事务。 对于创建对象仓库和索引的操作，只能在版本更新事务内进行，因为API不允许数据库中的数据仓库在同一个版本中发生变化, 所以当我们创建或删除数据仓库时，必须使用新的版本号来更新数据库的版本，以免重复修改数据库结构。

dbConnect.onupgradeneeded = (e) => {
    e.target.result
    //为一个IDBDatabase对象，代表连接成功的数据库对象
    //e.target.transaction代表IDBTransaction事务对象，此处代表版本更新事务
}
//指定的版本号大于数据库当前版本号时触发

创建对象仓库

let store = idb.createObjectStore(name, option)
//第一个参数为一个字符串, 代表对象仓库名
option = {
    keyPath: ;
    //相当于mongodb的_id，值在插入数据时对象的一个键名
    autoIncrement: ;
    // 相当于_id自动增加
}

创建索引

let name = "userNameIndex"
let keyPath = "username"
let option = {
    unique: ;
    multiEntry: ;
}
idx = store.createIndex(name, keyPath, option)

使用事务

let storeNames = ["Users"];
//对象仓库名，可以是字符串或数组

let mode = "readonly"
//只读事务

let mode = 'readwrite'
//读写事务

let tx = idb.transaction(storeNames, mode);
//开启事务，返会IDBTransaction对象

idb.objectStoreNames
//返回所有对象仓库名构成的数组

注：用于开启事务的transaction方法必须被书写到某一个函数中去，而且事务将在函数结束时被自动提交，不需要显示提交事务，必要时可以使用tx.abort()中止事务
